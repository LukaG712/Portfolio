Database

Ontwerp en implementatie van een relationele databank met CRUD-functionaliteit.

Over dit project

In dit project ontwierp en implementeerde ik een relationele databank met MySQL. Ik maakte een ER-diagram, normaliseerde de tabellen tot derde normaalvorm en schreef SQL-queries voor alle CRUD-operaties (Create, Read, Update, Delete).

De databank werd gekoppeld aan een eenvoudige webinterface zodat gebruikers data kunnen beheren via een browser zonder direct SQL te moeten schrijven.

Periode

Academiejaar 2023–2024

Technologieen

MySQL SQL PHP ER-diagram

Wat heb ik geleerd?

Database-ontwerp vereist veel vooruitdenken. Door te normaliseren vermijd je redundante data en inconsistenties die later problemen geven. Ik leerde het verschil tussen 1-op-1, 1-op-veel en veel-op-veel relaties in de praktijk toepassen en hoe je die vertaalt naar tabellen met foreign keys.

Het schrijven van complexe SELECT-queries met JOINs, WHERE-condities en aggregatiefuncties zoals COUNT() en SUM() gaf me inzicht in hoe krachtig SQL is als querytaal. Ik leerde ook hoe je PHP gebruikt als brug tussen de database en de gebruiker.

Uitdagingen

Het normaliseren van de databank was in het begin verwarrend. Bepalen welke gegevens in een aparte tabel thuishoren en hoe je die correct koppelt met foreign keys vroeg meerdere iteraties. SQL-injectie voorkomen via prepared statements was ook een belangrijk veiligheidsaspect dat aandacht vergde.
